Configuration · Premiers pas

Configurer Claude pour votre entreprise

Pas de compte, pas de configuration — et vous travaillez dans le flou. En 30 minutes, vous installez Claude de façon sécurisée, efficace, et adaptée à votre façon de travailler.

Étape 1 — Choisir la bonne version
1

Quelle version choisir ?

La version détermine ce que vous pouvez envoyer à Claude

Claude gratuit

Données publiques uniquement. Pas de DPA. Ne jamais envoyer de données clients, RH ou financières.

Claude Pro Pour vous

Priorité réseau, projets, mémoire. Avec le DPA signé → données clients autorisées sous conditions.

Claude for Work (Teams)

Usage équipe, DPA inclus, hébergement EU. Pour les structures avec plusieurs utilisateurs.

Action immédiate si vous êtes sur Claude Pro

Allez dans Settings → Legal & Compliance → signez le Data Processing Agreement. 5 minutes. Sans cette étape, vous n'avez pas de base légale pour les données clients même avec Claude Pro.

Étape 2 — Vos instructions personnalisées (racine)
2

Configurer votre contexte une fois pour toutes

Ces instructions s'appliquent à toutes vos conversations

Les instructions personnalisées sont votre "carte de visite" que Claude lit avant chaque conversation. Vous n'avez plus à vous présenter à chaque session.

1
Claude.ai → cliquez sur votre avatar en bas à gauche → Paramètres
2
Onglet Profil → section "Instructions personnalisées"
3
Collez le template ci-dessous, complétez les zones entre [ ]
Template — instructions personnalisées racine
Contexte de mon entreprise Je suis [votre prénom], dirigeante de [nom de l'entreprise]. Secteur : [votre secteur d'activité]. Taille : [nombre de collaborateurs]. Ma région : [ville ou région]. Comment je travaille Je préfère des réponses : [courtes et directes / détaillées / avec exemples]. Mes principaux usages de Claude : [ex : rédaction emails, analyse documents, préparation RDV]. Outils que j'utilise au quotidien : [ex : Google Workspace, Notion, etc.]. Mes règles de sécurité (toujours actives) Si je t'envoie un document et que tu détectes des données sensibles (noms de clients, IBAN, données RH, données de santé), signale-le avant d'aller plus loin. Ne traite jamais de données confidentielles que je t'aurais envoyées par erreur sans m'en avertir. Quand je tape /secure-guard suivi d'un contenu : Analyse si ce contenu contient des données sensibles RGPD. Verdict : 🟢 Sûr / 🟡 À anonymiser (liste quoi) / 🔴 Ne pas envoyer (explique pourquoi). Si 🟡 : propose une version avec les données remplacées par [PRÉNOM], [MONTANT], etc.
Étape 3 — Créer votre Projet Gouvernance IA
3

Votre projet dédié — mémoire et skills

Claude se souvient de tout ce qui est dans ce projet

Un Projet Claude, c'est un espace où vos conversations s'accumulent et où vous pouvez définir des comportements spécifiques (vos skills). Claude voit tout l'historique du projet à chaque nouvelle conversation.

1
Claude.ai → barre latérale gauche → Nouveau projet → nommez-le Gouvernance IA [votre entreprise]
2
Ouvrez le projet → Instructions du projet → collez vos skills (amélioration continue, veille)
3
Désormais, travaillez dans ce projet pour tout ce qui concerne votre gouvernance IA — Claude accumule le contexte
Créez un second projet "Veille IA" séparé

Gardez la veille dans un projet dédié. Comme ça, dans 6 mois Claude peut comparer : "En janvier vous surveilliez X, maintenant voici ce qui a changé."

Étape 4 — Votre premier test
4

Tester que tout fonctionne

3 tests à faire maintenant, avant de fermer cette page
A
Test Secure Guard — dans n'importe quelle conversation Claude, tapez :
/secure-guard Bonjour, voici la facture de M. Dupont, IBAN FR76 1234 5678 9012, montant 1 450 €.
Claude doit répondre 🟡 et proposer une version anonymisée.
B
Test contexte — démarrez une nouvelle conversation et demandez :
Qui suis-je et dans quel secteur je travaille ?
Claude doit répondre avec vos informations sans que vous les ayez redonné.
C
Test projet — depuis votre projet Gouvernance IA, tapez :
/amélioration-continue
Si le skill est bien installé, Claude lance l'analyse. S'il répond "je ne comprends pas cette commande", revérifiez que le skill est bien dans les instructions du projet.
Étape 5 — Votre prompt de démarrage universel

Quand vous ne savez pas quoi demander

Ce prompt s'utilise en début de session pour demander à Claude de vous proposer comment il peut vous aider — en partant de votre contexte.

Prompt universel de démarrage — à utiliser quand vous ouvrez Claude
Sur la base de ce que tu sais de mon profil et de mon activité : 1. Qu'est-ce que tu pourrais faire pour moi aujourd'hui qui m'économiserait du temps ? 2. Y a-t-il quelque chose que tu remarques dans ma façon de travailler avec toi que je pourrais améliorer ? 3. Propose-moi une tâche concrète à te déléguer maintenant, que je n'aurais probablement pas pensé à te confier. Sois direct. Propose des choses faisables en moins de 20 minutes.

Checklist de configuration — cochez au fur et à mesure

Version choisie — Claude gratuit, Pro ou Work selon vos usages
DPA signé — Settings → Legal & Compliance (si Claude Pro ou Work)
Instructions personnalisées — profil + contexte + skill /secure-guard installés à la racine
Projet "Gouvernance IA" créé — avec skill /amélioration-continue dans les instructions
Projet "Veille IA" créé — avec skill /veille hebdo et /veille mensuel
3 tests effectués — Secure Guard, contexte, amélioration continue
Réflexe DDO enseigné à l'équipe — Département → Donnée → Outil avant chaque usage IA
